**行號：** `main_pipeline.py` 第 169 行

**問題：** `main_pipeline.py` 調用不存在的方法

classification_result = self.pose_classifier.classify_pose_sequence(pose_json_path)

### ❌ 錯誤 4：OpenAI API 過時語法
**檔案：** `3_semantic_advisor.py`  
**行號：** 第 200 行

**現狀：**
```python
response = self.openai_client.ChatCompletion.create(
    model=model,
    messages=[...],
    max_tokens=max_tokens,
    temperature=temperature
)
```

**問題：** 使用舊版 OpenAI API 語法，與 `requirements.txt` 的 `openai>=1.20.0` 不兼容

**修正方式：**
```python
# 需要改為新版語法
response = self.openai_client.chat.completions.create(
    model=model,
    messages=[...],
    max_tokens=max_tokens,
    temperature=temperature
)
```

## 🟡 優先級 2：邏輯錯誤（資料處理錯誤）

### ❌ 錯誤 6：資料庫寫入結構不匹配
**檔案：** `main_pipeline.py` 與 `4_database_handler.py`  
**行號：** `main_pipeline.py` 第 195 行

**問題：** 資料庫期望的資料結構與 `SemanticAdvisor` 返回的結構不同

**現狀：**
```python
# main_pipeline.py
advice_result = self.semantic_advisor.generate_advice(classification_result)
self.database_handler.insert_pose_result(advice_result)
```

**database_handler.py 期望的結構（第 201-203 行）：**
```python
video_info = result_data.get('video_info', {})
classification = result_data.get('classification_summary', {})
metadata = result_data.get('metadata', {})
```

**問題：** `advice_result` 的結構包含 `classification_summary` 而非 `classification_result`

**修正方式：** 調整資料結構，或在 `insert_pose_result()` 增加資料結構轉換

---

### ❌ 錯誤 7：缺少分類結果路徑
**檔案：** `4_database_handler.py`  
**行號：** 第 215-216 行

**現狀：**
```python
result_data.get('classification_json_path', ''),  # classification_json_path
result_data.get('advice_json_path', ''),  # advice_json_path
```

**問題：** `SemanticAdvisor.generate_advice()` 沒有設置這些路徑

**修正方式：** 在 `SemanticAdvisor` 生成建議時，需要記錄所有相關檔案路徑

---

### ❌ 錯誤 8：返回資料缺少必要欄位
**檔案：** `3_semantic_advisor.py` 的 `generate_advice()` 方法

**問題：** 返回的資料字典缺少 `video_info` 等欄位，導致資料庫寫入失敗

**修正方式：** 在 `generate_advice()` 中，需要保留原始的 `classification_result` 資料並整合進返回結果
