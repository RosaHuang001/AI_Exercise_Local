🫀 AI 智慧心衰復健系統 (AI_Exercise_Local)
結合 ACSM 臨床指引與 YOLO+RAG 技術的個人化復健方案
本系統專為心臟衰竭 (Heart Failure) 患者設計，旨在解決傳統居家復健中「安全性難以監測」與「動作準確度缺乏回饋」的痛點。

🏥 核心醫學邏輯 (Medical Logic)
本系統之運動處方與安全過濾邏輯嚴格遵循 ACSM (美國運動醫學學會) 臨床指引：

生理數據硬過濾 (Hard Filter)：

系統即時分析收縮壓 (>180 mmHg)、舒張壓 (>110 mmHg) 與安靜心跳 (>120 bpm)，判定使用者是否處於穩定狀態。

若數值異常或出現呼吸困難、胸痛等症狀，系統將自動攔截運動計畫並建議就醫。

NYHA 分級動態強度建議：

Class I-II：建議 RPE 指標控制在 [RPE:11-13] (微喘但可對話)。

Class III (重點保護)：強制限制運動強度於 [RPE:11] 以下，避免過度心肺負荷。

Borg RPE 感官知覺指標：

透過 AI 摘要系統產出具備醫病溝通溫度的 RPE 強度框，提供病患具體、白話的強度參考。

💻 關鍵技術特徵 (Technical Features)
1. RAG 知識檢索引擎 (Retrieval-Augmented Generation)
精準處方生成：利用 RAG 連結內部的 ACSM 知識庫（hf_chunks.json），確保 AI 生成的建議具備醫學根據，而非隨機生成的文字。

上下文感知：結合病患的年齡、體重、手術史等數據，透過 RAG 引擎過濾出最符合個體風險的運動動作。

2. YOLOv11 視覺分析與特徵降維
即時骨架糾錯 (Real-time Pose Feedback)：

顏色連動機制：當動作角度與標竿數值偏差 > 20% 時，骨架即時轉為紅色（角度不足）；達標時轉為綠色（正確）。

有效次數篩選：僅記錄骨架在綠色狀態下完成的完整動作，確保復健品質。

單角度投影算法：

針對居家環境優化，將示範影片的角度數據「投影」至二維平面，解決使用者僅有單一攝影機視角的問題。

物理負荷計算：

利用 YOLO 辨識結果計算關節相對衝擊力（倍體重），作為 RPE 指引生成的關鍵物理輸入。

📊 數據驅動的復健回饋 (Data-Driven Insights)
今日動作品質分析圖：

系統自動統計「標準動作(綠色)」與「角度不足(紅色)」的比例分佈。

疲勞預警與品質預測：

整合 NYHA 分級與生理指標，由 GPT 產出「品質預測」，提醒使用者在肌肉疲勞前適度休息，預防代償動作導致的受傷。

🚀 專案展示流程 (Demo Flow)
資料建立：輸入基本資料，系統根據 NYHA 分級呈現不預設選項的嚴謹評估。

生理監測：填寫今日血壓/心跳，啟動 ACSM 醫學過濾動畫。

智慧分析：產出專屬 RPE 指引、一週課表與品質分布預測圖。

即時跟練：開啟雙畫面，展現紅色/綠色骨架變色與頂部有效次數累計的即時比對技術。
